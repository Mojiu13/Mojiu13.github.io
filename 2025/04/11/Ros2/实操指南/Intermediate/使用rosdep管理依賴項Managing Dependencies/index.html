  • rosdep是什么
    • 是一个依赖管理实用程序，可以与软件包和外部库一起使用。它是一个命令行实用程序，用于识别和安装构建或安装软件包所需的依赖项
    • 它通常在构建工作空间之前被调用，用于安装工作空间内包的依赖项。
  • 关于package.xml
    • package.xml 是您的软件中 rosdep 找到依赖关系集的文件。重要的是 package.xml 中的依赖关系列表是完整和正确的，这样所有的工具才能确定软件包的依赖关系。缺少或不正确的依赖关系可能导致用户无法使用您的软件包，导致工作区中的软件包按照错误的顺序构建，以及软件包无法发布。
    • 在package.xml文件中的依赖项通常被称为“rosdep key。这些依赖项是由软件包的创建者手动填充到package.xml文件中的，应该是所需的任何非内置库和软件包的详尽列表。
    • 关于标签的说明
      • <_depend>
        • 这些是应该在构建时间和运行时间为您的软件包提供的依赖项。对于 C++ 软件包，如果有疑问，请使用此标签。纯 Python 软件包通常没有构建阶段，因此不应该使用这个，而应该使用<exec_depend>。
      • <_build_depend>
        • 如果您只在构建软件包时使用特定的依赖项，而不是在执行时使用，您可以使用<build_depend>标签。
      • <_build_export_depend>
        • 如果您导出一个包含来自依赖项的头文件的头文件，那么其他依赖于您的包的包将需要它。这主要适用于头文件和 CMake 配置文件。您导出的库包引用的库通常应该指定<depend>，因为它们在执行时也是必需的。
      • <_exec_depend>
        • 此标签声明了在运行您的软件包时所需的共享库、可执行文件、Python 模块、启动脚本和其他文件的依赖关系。
      • <_test_denpend>
        • 这个标签声明仅测试需要的依赖项。这里的依赖项不应该与由<build_depend>、<exec_depend>或<depend>指定的键重复。
  • rosdep工作原理
    • rosdep 将检查其路径中的 package.xml 文件，或特定软件包，并查找其中存储的 rosdep -key。然后，这些键将与中央索引进行交叉引用，以在各种软件包管理器中找到适当的 ROS 软件包或软件库。最后，一旦找到软件包，它们就会被安装并准备就绪！
    • rosdep 通过在本地机器上检索中央索引来工作，这样每次运行时就不必访问网络（在 Debian/Ubuntu 上，其配置存储在 /etc/ros/rosdep/sources.list.d/20-default.list 中）。
    • 中心索引被称为rosdistro，可以在网上找到。我们将在下一节中更深入探讨。
  • 配置package.xml
    • 如果您的软件包依赖的软件包是基于 ROS 的，并且已经发布到 ROS 生态系统 1 中，例如nav2_bt_navigator，您可以简单地使用软件包的名称。您可以在https://github.com/ros/rosdistro中找到所有已发布的 ROS 软件包列表，位于<distro>/distribution.yaml（例如humble/distribution.yaml）适用于您所使用的 ROS 发行版。
    • 如果您想依赖于一个非 ROS 软件包，通常被称为“系统依赖项”，您将需要找到特定库的密钥。一般来说，有两个感兴趣的文件：
      • rosdep/base.yaml 包含 apt 系统依赖
      • rosdep/python.yaml 包含了 Python 依赖项
    • 要找到一个key，搜索这些文件中的库并找到名称。这是放在package.xml文件中的key。（在rosdep/base.yaml中搜索）

  • 如何使用rosdep工具
    • 安装
      • apt-get install python3-rosdep
    • 初始化
      • sudo rosdep init
      • rosdep update
    • 使用：
      • rosdep install –from-paths src -y –ignore-src
      • --from-paths src 指定要检查的路径，以解析键的 package.xml 文件。
      • -y 表示默认为所有提示从软件包管理器安装而无需提示
      • --ignore-src 意味着忽略安装依赖项，即使存在 rosdep 键，如果包本身也在工作区中。
